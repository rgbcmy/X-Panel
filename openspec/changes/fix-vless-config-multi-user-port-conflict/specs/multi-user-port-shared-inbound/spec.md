## 新增需求

### 需求：共享端口的入站回退查找
当为某用户配置入站时，若其 `vless_config.inbound_port` 已被另一用户的入站占用，同步任务**必须**将该用户作为客户端加入已有的端口匹配入站，而不是尝试创建新的入站。

#### 场景：第二个用户与第一个用户共享同一端口
- **当** 用户 A 已触发在端口 P 上创建了一个入站
- **当** 用户 B 的 `vless_config.inbound_port` = P 且拥有不同的 `inbound_tag`
- **则** 同步任务**必须**找到端口 P 上已有的入站
- **则** 同步任务**必须**将用户 B 作为客户端加入该入站

#### 场景：用户已是端口匹配入站的客户端
- **当** 端口 P 上存在一个已包含用户 B 的入站
- **当** 再次接收到用户 B 的同步数据
- **则** 同步任务**必须**更新用户 B 在该入站中的现有客户端条目（UUID、速度限制、flow）

#### 场景：请求端口上不存在任何入站
- **当** 没有已有入站匹配 `vless_config.inbound_port`
- **则** 同步任务**必须**按原有逻辑在该端口上创建新入站

### 需求：每次同步周期为所有带 vless_config 的用户完成配置
同步任务**必须**成功为每一个携带 `vless_config` 的用户完成配置，无论有多少用户共享相同的 `inbound_port`。

#### 场景：N 个用户共享同一端口
- **当** N 个用户均拥有相同的 `vless_config.inbound_port`
- **则** 同步任务**必须**使 `processedCount` = N 且 `errorCount` = 0
- **则** 全部 N 个用户**必须**以客户端身份出现在唯一的共享入站中
